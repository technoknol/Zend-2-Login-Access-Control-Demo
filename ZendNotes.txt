Zend Notes:
===========
Main (app start):
- /public/index.php calls Zend/Mvc/Application each time a non-static file
is requested by the user.

- Project Structure:
/config - Application-level configuration files
/config/autoload - Application-level configuration files
/config/application.config.php is loaded by Zend\Mvc\Service\ServiceManagerConfig
/data - Data generated by the application, for example caches
/module - The code of the application, organized in modules
/public -> What is going to be served
/vendor -> Composer dependencies of the project

- Module:
-- All modules must be registered in /config/application.config.php
-- Modules should be created with proper dir structure:
--- http://framework.zend.com/manual/current/en/modules/zend.mvc.intro.html
-- The Zend\ModuleManager loads the module (i.e., class)
-- If a module has an init() function, it will be called by the Zend\ModuleManager

- Debug
-- To install XDebug, go to http://xdebug.org/wizard.php and follow instruction
-- Retrieve your IPv4 address with ipconfig in DOS session
-- Edit php.ini and add:
zend_extension = "C:\php\php_xdebug-2.3.3-5.5-vc11.dll"
xdebug.remote_autostart=on
xdebug.remote_enable=on
xdebug.remote_handler=dbgp
xdebug.remote_host=192.168.1.19
xdebug.remote_port=9000
xdebug.remote_mode=req
xdebug.idekey="netbeans-xdebug"
-- Double-check the idekey and port value in Netbeans PHP configuration
-- Reboot Netbeans
